Цели:
----- 
1) изучить прототокол IPFS;
2) исследовать, насколько кеширование и прямая передача данных влияет на эффективность вычислений;
3) убедиться, что IPFS эффективно реализует кэширование и прямую передачу данных.

Задачи
------ 
Требуется исследовать зависимость ускорения от числа разбиений информационного массива в задаче
нахождения ортогональных пар диагональных латинских квадратов на разных конфигурациях распределенных систем:
1)   данные передаются каждый раз с сервера Everest для новой задачи;
2)   данные берутся из распределенной файловой системы (Windows Server);
3)   весь массив данных находится локально на каждом узле;
4)   в клиенте Everest настроено автоматическое кэширование данных (по-возможности, прямая передача данных между узлами);
5)   данные передаются по протоколу IPFS.

Методика
--------
1. Первый эксперимент выполняется для целого массива (запускается одна задача). Полученное время вычисления задачи берется как базовое для расчета ускорения.
2. Во втором эксперименте массив делится на 2 подмассива, в третьем -- на 3 и так далее. Фиксируется время вычислений и ускорение относительно первого эксперимента.
3. Процесс п.2 завершается, когда на очередном разбиении время сравняется или будет больше чем в первом эксперименте (не будет наблюдаться ускорение из-за высоких накладных расходов на управление вычислениями).
4. Время вычислений для каждого разбиения массива усредняется по 3 экспериментам (если возникнет большой разброс значений в 3-х экспериментах, придется скорректировать методику и брать больше значений для усреднения).

Ожидаемые результаты
--------------------
1. Ожидаемый худший случай - (1). Ожидаемый лучший случай - (3). Другие результаты должны быть между (1) и (3).
2. IPFS сравнивается со "штатной" распределенной файловой системой для исследуемой конфигурации -- Windows Server, ожидается лучшее поведение, за счет того, что в IPFS нет поддержки блокировок при совместном доступе и есть прямая передача между узлами.
3. IPFS сравнивается с интегрированными в Everest механизмами (кэширование, прямая передача между узлами), ожидается похожее поведение.
